Assignment Introduction: Provisioning and Deployment

1 About This Text

This text introduces the Provisioning and Deployment assignment.

2 Introduction

A vital part of making the best use of cloud resources is to have a controllable way to set up your cloud environment. Ironically, a large part of this consists of not using cloud resources but rather configure a transparent way to set up a local development environment, a potentially local test environment, and the production environment. The goal is to be able to do this with the same command(s) and to make sure that all environments are similarly configured.

The first goal of this assignment is for you to get hands on experience in setting up these different enviromments in a transparent and repeatable way. You will install a basic toolset for running virtual machines to get started with this.

Provisioning can be done with different levels of automation, ranging from written instructions that are manually executed, and all the way up to a scripted installation of e.g. a microservice.

The second goal of this assignment is for you to get hands on experience using different provisioning methods starting with simple shell commands and continuing with a provisioning tool.

These building blocks are then used toghether to configure a one-click setup of several machines with different roles, and ultimately deploy these to a cloud provider.

The assignment consists of two tracks: One practical track where you actually set up your machines, and one reflective track where you log what you have done and reflect upon it. Think of this part as the lab notebook stabled to the desk in a chemics lab.

4 Experiential Learning

4.1 Reflective Assignment: Provisioning and Deployment Experience Report

Write a brief document that summarises your workflow in this assignment.

  • What did you do, and in what order.
  • What challenges did you encounter?
  • What did you to do solve the challenges?
  • What options did you have? Please explore each option to some depth and motivate why you did or did not select it.
  • What are the most important points to remember? What would you like to pass on to the developer that will take over your work once you get your next promotion?
  • What are the security implications e.g. of using the stock Ubuntu image provided by the Cloud Provider?

4.2 Practical Assignment: Provisioning and Deployment

The assignment is organised as a set of sprints, where each sprint is estimated to roughly 10h.

Name Comment
Getting Started with Virtualisation Introduces a tool for Virtualisation (VirtualBox) and a Layer on top of this that enables automation (Vagrant).
Provisioning Introduces how to set up your virtual machine with the software you need in order to run your applications.
Multi-Machine Setup Introduces how to configure startup of several machines with different roles at once.
Enter the Cloud Introduces how to seamlessly transfer your local machine setup to a cloud provider.

4.3 Assignment Submission

  • Submit your Experience report to the teachers for marking.
  • Book a time with a teacher to start up and demo your provisioned cloud environment.

Conditions of Satisfaction

When marking the assignment we are looking for the following:

  1. Are an appserver and a dbserver deployed from a single Vagrantfile?
  2. Are the appserver and dbserver provisioned with node.js and mysql as required?
  3. Can the appserver and dbserver be seamlessly deployed either as a local VM or as a DigitalOcean droplet?
  4. Can the student explain what they do in their deployment scripts?
  5. In the experience report:
    1. Are the encountered challenges briefly explained?
    2. Are the causes for the challenges explored in a couple of paragraphs?
    3. Are other sources (books, web or research articles) used to explain and explore the challenges?
    4. Are the solutions to the challenges explained and explored in a couple of paragraphs?
    5. Do the solutions make sense?
    6. Are options listed and described?
  6. In the experience report:
    1. Is there a list of key learning highlights (“most important points”)?
    2. Are these highlights briefly explained?
  7. Are security implications briefly explored in the experience report?

5 Acceptance Tests

You are done with this assignment when:

  • You have submitted your experience report to a teacher for review.
  • You have demoed your configuration to a teacher.
  • You have remembered to destroy your machines once you are done!
  • You have received a passing grade from the teacher.
